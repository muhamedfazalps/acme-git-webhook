# F5 Big-IP target

Uploads certificates to an F5 Big-IP via iControl REST and automatically updates matching Client SSL profiles.

## Configuration

```yaml
targets:
  - name: "f5-paris"
    provider: "f5"
    addr: "https://bigip.example.com"
    username: "admin"
    password_path: "/run/secrets/f5_password"
    verify: true
    timeout: 30
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `name` | `str` | **required** | Unique identifier for this target. |
| `provider` | `str` | `"f5"` | Must be `"f5"`. |
| `addr` | `str` | **required** | Base URL of the F5 iControl REST endpoint. |
| `username` | `str` | **required** | F5 admin username. |
| `password_path` | `str` | **required** | Path to a file containing the F5 password (mounted secret). |
| `verify` | `bool` | `true` | Whether to verify the F5 TLS certificate. |
| `timeout` | `int` | `30` | HTTP request timeout in seconds. |

## How it works

### 1. Upload certificate artifacts

The PEM fullchain and private key are uploaded to the F5 via:

- `POST /mgmt/tm/sys/file/ssl-cert` — uploads the fullchain as an SSL certificate object
- `POST /mgmt/tm/sys/file/ssl-key` — uploads the private key as an SSL key object

Object names are derived from the domain (e.g. `example.com` → `/Common/example.com`).

### 2. Discover matching SSL profiles

The target queries `GET /mgmt/tm/ltm/profile/client-ssl?$select=name,cert,key` and scans for profiles whose `cert` field contains:

- The bare domain name (e.g. `example.com`)
- The sanitized wildcard name (e.g. `wildcard.example.com` for `*.example.com`)
- An empty/`none` cert reference

### 3. Update profiles

Each matching profile is updated via `PUT /mgmt/tm/ltm/profile/client-ssl/{partition}/{name}` to reference the newly uploaded certificate and key.

## Wildcard domain handling

F5 object names cannot contain `*` characters. The target sanitizes the domain:

| Domain | Object name |
|--------|-------------|
| `example.com` | `example.com` |
| `*.example.com` | `wildcard.example.com` |

The `*` prefix is replaced with `wildcard.`. Profile matching then checks both the sanitized name and the original wildcard notation.

## Result

```json
{
  "target": "f5-paris",
  "provider": "f5",
  "status": "ok",
  "details": {
    "host": "https://bigip.example.com",
    "cert_name": "/Common/example.com",
    "key_name": "/Common/example.com",
    "updated_profiles": ["/Common/example.com_client_ssl"]
  }
}
```
