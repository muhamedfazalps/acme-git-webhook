# Cible : F5 Big-IP

Déploie les certificats sur un ou plusieurs équilibreurs F5 Big-IP via
l'API iControl REST.

## Configuration

```yaml
targets:
  - name: "f5-paris"
    provider: "f5"
    addr: "https://bigip.example.com"
    username: "admin"
    password_path: "/run/secrets/f5_password"
    verify: true
    timeout: 60
```

| Champ | Type | Défaut | Description |
|-------|------|--------|-------------|
| `name` | `str` | — | Identifiant unique de la cible |
| `provider` | `str` | — | `"f5"` |
| `addr` | `str` | — | URL de l'API REST du Big-IP |
| `username` | `str` | — | Compte administrateur |
| `password_path` | `str` | `"/run/secrets/f5_password"` | Chemin du fichier contenant le mot de passe |
| `verify` | `bool` | `true` | Vérification TLS |
| `timeout` | `int` | `60` | Timeout HTTP (secondes) |

## Fonctionnement

1. **Upload du certificat** — Envoie le fullchain PEM vers
   `/mgmt/tm/sys/file/ssl-cert` via iControl REST.
2. **Upload de la clé privée** — Envoie la clé privée vers
   `/mgmt/tm/sys/file/ssl-key`.
3. **Mise à jour des profils** — Détection automatique des profils
   Client SSL dont le champ `cert` contient le nom de domaine, puis
   mise à jour vers le nouveau certificat.

## Domaines wildcard

Pour les domaines wildcard (`*.example.com`), le nom est nettoyé
automatiquement pour être compatible avec les objets Big-IP :

- `*` est supprimé
- `.` est remplacé par `_`

Exemple : `*.example.com` → `wildcard_example_com`

## Exemple de déploiement

```bash
# Déploiement vers toutes les cibles F5 configurées
curl -X POST http://localhost:8000/deploy/example.com \
  -H "Authorization: Bearer sk-XXXXXXXXXXXX"

# Déploiement vers une cible F5 spécifique
curl -X POST http://localhost:8000/deploy/example.com/f5-paris \
  -H "Authorization: Bearer sk-XXXXXXXXXXXX"
```

## Vérification manuelle

```bash
# Vérifier le certificat sur le Big-IP
curl -sk -u admin:password \
  "https://bigip.example.com/mgmt/tm/sys/file/ssl-cert/~Common~example_com" \
  | jq '.certificate'
```
